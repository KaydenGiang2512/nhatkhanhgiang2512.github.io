OVERVIEW:  
In basic terms, Git serves as an open source distributed version control system which allows you to effortlessly store, modify, push and keep track of your files completely free of charge.

   Since its initial launch, Git has gained massive support from the developer community and gradually became the most popular version control system (VCS) out there.
   One of the biggest reasons for Git being favored over Subversion is the fact that Git represents a distributed VCS as opposed to Subversion, which is built as a centralized VCS (see the image above).

  - Some advantages of the DVCS: 
+ Ensures productivity even without a network connection;
+ Common proccesses (commits, adds, viewing history) becomes much faster because everyone can work on their local repositories instead of communicating with a central server;
+ Allows private work before publishing to the shared server;
+ Copies of each person's work also function as remote backups, so it prevents relying on a single physical machine.

So what else separates Git from other version control systems?

 Amazing performance

Convenient staging areas

Cheap local branching

Allowing multiple workflows

  Due to the superiority in processing speed and efficiency, along with other useful features compared to its counterparts (Subversion, CVS, Perforce, ClearCase), Git has made its way into the mainstream of DevOps tools and is being used by several top-tier Tech companies​ like Google, Facebook, Netflix, Twitter, etc.

BASIC COMMANDS

1) git config: This command is to set a git configuration value on a global/local project level.
Usage: git config - -global user.name “[name]”  
                    git config - -global user.email “[email address]”  
(These commands sets up your username/email address used for future commits.)
Examples:

2) git init: As the name implies, this command initializes a new git repository.
Usage: git init [repo name]
Example:

3) git clone: This command obtains a repository from an existing URL address.
Usage: git clone [URL]
Example:

4) git add: This is used to add a file or many files to the staging area.
Usage: git add [file] (add a file)
             git add * (add more than 1 file)
             git add . (Add all files)
Example:

5) git commit -m: This command acts like a record that stores the file permanently within the version history.
Usage: git commit -m "Type the commit message"
Example:

6) git commit -a: This command commits any file you have added and also files that have been changed since then.      
Usage: git commit -a 
Example:    

7) git push: This is a crucial command that sends files from your local workspace directly to your repository.
Usage: git push 
Example:

8) git pull: This command is used to retrieve file changes on your remote server to your working directory.
Usage: git pull [Link of your repo] 
Example:

9) git branch: This command list all exising local branches in the current repository.
Usage: git branch
             git branch [branch name] (This creates a new branch)
Examples:

10) git checkout: This command switches your directory from one branch to another.  
Usage: git checkout [branch name]
Example:

11) git merge: This command is used to merge the selected branch's history into your current branch. 
Usage: git merge [branch name]
Example:

12) git log: This command lists the entire version history of the current branch.
Usage: git log
Example:
